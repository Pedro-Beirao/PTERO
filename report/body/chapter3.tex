\chapter{State of the Art} \label{chap:state-of-the-art}

This chapter provides a comprehensive analysis of the current landscape of development environments for \ac{CPPS} and the \textit{IEC 61499} standard. It categorises and critically examines the existing development tools, while looking for possible gaps in the ecosystem.

\section{Ecosystem Summary}

Currently, the development ecosystem is composed only by desktop-based \acp{IDE}. These tools are typically monolithic applications installed locally on engineering workstations.

\subsection{Open-Source}
The most prominent open-source implementation is the \textit{Eclipse 4diac} \ac{IDE} (based on the \textit{Eclipse} framework) which supports a variety of \acp{RTE} such as \textit{4diac FORTE} and \textit{DINASORE}. This \ac{IDE} is very feature rich at the cost of significant system resources and increased complexity, resulting in a steep learning curve, which makes it more difficult to use \cite{weber_usability_2019}.

An alternative is \textit{Jetbrains FBME} which is a much simpler modelling environment \cite{sorokin_novel_2022}. While easier to use than 4diac \ac{IDE}, it still is a local-first development environment. The importance of this will be explained in \Cref{sec:localfirst}.

\subsection{Commercial}
The commercial side is occupied by \textit{EcoStruxure Automation Expert}, \textit{ISaGRAF Workbench} and other development environments that usually have their own built-in proprietary \ac{RTE}. This makes them often very complete production ready tools, but they are not very flexible and may not be the best fit for many needs.

\section{Evolution of Web-Based Engineering Tools}

The software engineering domain has seen a massive shift from desktop \acp{IDE} to cloud-native environments. This trend is now influencing the domain of \ac{ICS}, and while recent studies have proposed a \ac{SOA} to integrate \textit{IEC 61499} with cloud layers \cite{torres_literature_2025}, the engineering tools themselves remain largely desktop-bound.

\section{Gap Analysis}
Current \textit{IEC 61499} tools face significant limitations that hinder modern \ac{CPPS} development:

% TODO need to rewrite these. make it a bigger list
\begin{itemize}
    \item \textbf{Platform Dependency:} Existing solutions are monolithic desktop applications, lacking the accessibility and portability of web-based environments.

    \item \textbf{Resource Intensity:} Current options are heavyweight and complex, creating a high barrier to entry compared modern development tools of other purposes.

    \item \textbf{Web Trends:} Despite industry trends, there is currently no mature, lightweight web interface for \ac{FBD}, leaving \ac{ICS} engineering behind the curve of cloud-native software development.
\end{itemize}
