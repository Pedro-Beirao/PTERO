\chapter{Introduction} \label{chap:introduction}

\section{Context}
Modern industrial production increasingly relies on \acp{CPPS} \cite{monostori_cyber-physical_2014}, which tightly couple computational intelligence with physical manufacturing processes to enable adaptive, data-driven operation. Engineering these systems requires a diverse set of software approaches, among which \ac{FBD} \cite{cho_function_1993} has become widely adopted in industrial automation. Rather than expressing control logic as lines of code, \ac{FBD} represents it as a network of interconnected graphical blocks, each encapsulating a discrete function. This visual paradigm offers tangible advantages: the structure of a program is immediately apparent, individual blocks can be reused across projects and the relationships between system components are explicit rather than implied. For these reasons, \ac{FBD} is a natural fit for controlling the machinery and processes found in manufacturing environments, often in conjunction with \acp{PLC} \cite{pinto_digital_2023, pang_formal_2015}.

Despite these advantages, \ac{FBD} introduces tooling demands that text-based languages do not. Constructing and editing a function block network requires specialized software capable of managing graphical composition, typed port connections and deployment configuration, none of which are concerns in a conventional code editor. The overhead this imposes has been documented in both industrial and academic literature, with studies noting that the complexity of graphical \ac{FBD} workflows compares unfavourably to the relative directness of text-based approaches \cite{fischer_measuring_2021}.

% Context IEC 61499
In industrial automation, it is common to use the \textit{IEC 61499} standard, which defines an event-driven \ac{FBD} approach for designing control software \cite{sorokin_novel_2022}. Unlike traditional \ac{PLC} programming languages, \textit{IEC 61499} emphasizes modularity, reusability and explicit separation of events and data, allowing developers to design complex distributed systems in a scalable and maintainable way. Function blocks define both algorithms and interfaces for inputs, outputs and events making them ideal for \ac{CPPS} development \cite{garcia_engineering_2017}. This approach is particularly suitable for distributed automation systems, where control logic must respond to events generated by multiple devices or sensors in real time. An example of such a function block interface is shown in \Cref{fig:function-block-interface}, highlighting the event inputs, outputs and data connections that enable flexible interaction between system components.

\acp{RTE} provide the underlying execution layer for \textit{IEC 61499} applications, responsible for running function blocks on target devices in accordance with the standard’s event-driven execution model.  In distributed \ac{CPPS} environments, \acp{RTE} are deployed across multiple devices, enabling the execution of decentralized control logic while maintaining coordination between system components.

Conversely, \acp{IDE} are responsible for the design, configuration and deployment of applications into \ac{CPPS} infrastructures. In this role, they act as the primary interface between the engineer and the distributed runtime system, bridging the gap between high-level system modelling and low-level execution. \acp{IDE} typically provide a graphical \ac{UI} like shown in \Cref{fig:4diac-ide}, with specialized tools for constructing function block networks, configuring device mappings and managing deployment workflows. \Cref{fig:ide-rte} represents the communications between these two layers.

\section{Motivation}
The evolution of \acp{CPPS} has increased the demand for flexible and intelligent control solutions capable of integrating physical processes with digital technologies. The \textit{IEC 61499} standard offer a structured, modular approach to building such distributed automation systems. It provides not only a technically robust foundation for industrial automation, but also a structured way of reasoning about distributed control systems. To fully leverage these benefits, development tools must evolve alongside industrial needs, supporting more accessible and scalable workflows.

\section{Problem}
Existing \textit{IEC 61499} development environments lack modern capabilities such as remote access, real-time collaboration and seamless integration with cloud infrastructures. These limitations hinder efficient development, maintenance and deployment of control applications within distributed \acp{CPPS}. As a result, current tools fall short in supporting the flexibility and scalability required by certain modern industrial automation systems. Besides this, the complexity of these current \acp{IDE} rules them out for many simpler use cases where the time wasted on setting up and learning them is not justifiable.

\section{Research Questions}
This research seeks to investigate how modern web and distribution technologies can be leveraged to enhance the development, deployment and operation of \textit{IEC 61499}-compliant systems. This study is guided by the following main research question:
\begin{quote}
How can a web-based \ac{IDE} support the modelling, deployment and execution of \textit{IEC 61499} applications?
\end{quote}

\noindent To further refine the scope, the following sub-questions are proposed:
\begin{quote}
To what extent can collaborative web technologies improve \textit{IEC 61499} engineering workflows compared with traditional desktop \acp{IDE}?
\end{quote}

\begin{quote}
What are the performance and usability implications of adopting a web-based collaborative architecture for \textit{IEC 61499} engineering?
\end{quote}

\section{Contributions}
This dissertation contributes to the field of \textit{IEC 61499} engineering by investigating the applicability of modern web technologies to the development of distributed automation systems. The main contributions of this work are:

\begin{itemize}
\item The design and implementation of \textit{PTERO}, a web-based \ac{IDE} for the development, deployment and monitoring of \textit{IEC 61499} applications.

\item The development of an architecture that integrates modelling, function block management, deployment and monitoring within a single browser environment, reducing the fragmentation commonly found in existing workflows.

\item The incorporation of real-time collaborative editing capabilities, enabling multiple users to work simultaneously on the same application.

\item An analysis of the benefits and limitations of adopting a web-based collaborative architecture for \textit{IEC 61499} engineering, providing insights for future research and development in this area.
\end{itemize}
